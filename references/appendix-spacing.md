# 附属書B-E: 文字間の空き量・分割可否・行調整

出典: W3C JLREQ (Requirements for Japanese Text Layout)
https://www.w3.org/TR/jlreq/

---

## 附属書B: 文字間の空き量の表の見方

文字間の空き量は、隣接する2つの文字クラスの組み合わせで決まる。
基本的な空き量の種類:

- **ベタ組み (0アキ)**: 文字間にアキなし。基本はベタ組み。
- **アキあり**: 文字間にアキを挿入。通常は和字幅の1/2または1/4。
- **1/4アキ**: 和字間隔の1/4の幅（例: 始め括弧の前）。
- **1/2アキ**: 和字間隔の1/2の幅（例: 句点・読点の後）。

空き量の原則:
1. 和文文字同士の基本間隔はベタ組み（アキなし）
2. 約物（cl-01〜cl-07）の前後にアキが入る場合がある
3. 和欧文間にはアキを入れる（cl-27: 通常1/4em）
4. アキ量は行調整時に詰め・空けの対象となる

表の読み方:
- 行: 前の文字のクラス（左にある文字）
- 列: 後の文字のクラス（右にある文字）
- 交差するセル: その組み合わせで挿入するアキ量

## 附属書C: 文字間での分割の可否の表の見方

行分割（禁則処理）は隣接する文字クラスの組み合わせで判断する。

分割の種類:
- **分割可**: その位置で行を折り返せる
- **分割不可**: その位置で行を折り返せない（禁則）
- **分割可（推奨しない）**: 折り返せるが望ましくない

主要な分割禁止の組み合わせ:
1. 行頭禁則文字の前で分割しない（cl-02, cl-04, cl-05, cl-06, cl-07の前）
2. 分離禁止文字（cl-08）と次の文字の間で分割しない
3. 繰返し記号（cl-09）と前の文字の間で分割しない
4. 長音記号（cl-10）と前の文字の間で分割しない
5. 拗音・促音と前の文字の間で分割しない（推奨）
6. ルビ付き文字の途中で分割しない（推奨）

表の読み方:
- 行: 行末側にくる文字のクラス
- 列: 行頭側にくる文字のクラス
- 交差するセル: 分割可否を示す

## 附属書D: 行調整で詰める処理が可能な箇所

行末を揃えるため（ジャスティフィケーション）、文字間を詰める場所:

### 詰め可能な箇所（優先順位順）:
1. **和欧文間 (cl-27)**: 最も詰めやすい。完全に消去可能。
2. **行末のアキ**: 句読点・括弧類の後のアキを詰める。
3. **始め括弧の前のアキ (cl-01前)**: 1/4アキを詰める。
4. **終わり括弧の後のアキ (cl-02後)**: 1/2アキを詰める。
5. **句点後のアキ (cl-06後)**: 1/2アキを詰める。
6. **読点後のアキ (cl-07後)**: 1/2アキを詰める。
7. **中点の前後のアキ (cl-05前後)**: 詰め可能。
8. **区切り約物の前のアキ (cl-04前)**: 詰め可能。

### 詰めてはいけない箇所:
- ベタ組みの和文文字間（アキがないため詰めようがない）
- 行頭禁則を生じさせる位置
- 読みやすさを損なう位置

## 附属書E: 行調整で空ける処理が可能な箇所

行末を揃えるため、文字間を空ける場所:

### 空け可能な箇所（優先順位順）:
1. **和欧文間 (cl-27)**: 最も空けやすい。1/2em程度まで拡張可能。
2. **和字間隔 (cl-21)**: 全角スペースをさらに空ける。
3. **始め括弧の前のアキ (cl-01前)**: 1/4アキをさらに空ける（最大1/2アキまで）。
4. **終わり括弧の後のアキ (cl-02後)**: 1/2アキをさらに空ける。
5. **句点後のアキ (cl-06後)**: 1/2アキをさらに空ける（最大1アキまで）。
6. **読点後のアキ (cl-07後)**: 1/2アキをさらに空ける。
7. **中点の前後のアキ (cl-05前後)**: 空け可能。
8. **区切り約物の前のアキ (cl-04前)**: 空け可能。

### 空けてはいけない箇所:
- 和文文字のベタ組み間（空けると読みやすさが損なわれる）
- 単語の途中（特に欧文単語の途中）

---

## 行調整の実践的なポイント

1. **優先順位**: 詰める場合も空ける場合も、和欧文間が最も自由度が高い。
2. **句読点のぶら下げ**: 句点・読点を版面外にはみ出させることで、行末揃えの負担を軽減。
3. **均等割り付け**: 最終手段として、行内のすべての文字間を均等に空ける方法もあるが、読みやすさが損なわれる。
4. **和文混植の注意**: 欧文が多い行は和欧文間の調整で対応し、和文が多い行は句読点周りの調整で対応する。
