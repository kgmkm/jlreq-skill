# JLREQ 第1章 序論（Introduction）

> AI エージェント向け要約。原文: https://www.w3.org/TR/jlreq/#intro

## 1. 文書の目的

- 日本語組版の処理方法を体系的にまとめた文書
- 主にテキスト組版（行の組み方）を対象とし、ページ全体のレイアウトも一部含む
- W3C の CSS 仕様や XSL-FO など、組版仕様を策定する際の基盤資料となる
- 「日本語組版の処理方法」（W3C 日本語組版タスクフォース作成）に基づく

## 2. 作成背景

- **JIS X 4051**（日本語文書の組版方法）を主要な基盤として作成
- ただし JIS X 4051 だけでは網羅できない事項を補完
- 実務上の組版慣習や、電子出版での実践も反映
- JIS X 4051:2004 が 2019 年に廃止されたため、本 JLREQ が実質的な後継資料

## 3. 基本原則

### 3.1 横組みと縦組み
- 日本語組版は **横組み** と **縦組み** の二つの基本方向がある
- 各方向で文字の配置、約物の処理、ルビの方向等が異なる

### 3.2 字幅の概念
- **全角文字**: 正方形（1em × 1em）を基本単位とする
- **半角文字**: 全角の幅の半分（0.5em）
- 文字サイズに対する相対的な配置が組版の基礎

### 3.3 行間と行のそろえ
- 行間は行の基準線間の距離で定義
- 和欧混植では和文文字と欧文文字のベースライン調整が必要

## 5. 主要用語参照

| 用語 | 定義（要約） |
|---|---|
| **字幅** | 文字の送り幅（全角=1em、半角=0.5em） |
| **行長** | 一行に配置する文字数または長さ |
| **行間** | 隣接する行の基準線間の距離 |
| **版面** | ページ内に文字・図版を配置する領域 |
| **行送り方向** | 文字を並べる方向（縦組=右→左、横組=上→下） |
| **字詰め方向** | 一行内に文字を詰める方向（縦組=上→下、横組=左→右） |
| **約物** | 句読点、括弧、ダッシュ類等の総称 |
| **和欧混植** | 和文文字と欧文文字を混在させる組版 |

## 6. AI エージェント実運用メモ

- 組版判断時はまず **縦組み/横組み** を確認する
- **字幅の扱い**（全角・半角・プロポーショナル）が多くのルールの前提
- **禁則処理**（第4章参照）は組版品質に直結する最重要項目
- **ルビ・圏点**（第5章参照）は小説組版で頻出
- JIS X 4051 廃止後は本 JLREQ が実質標準。CSS の `text-spacing`、`line-break` 等の策定にも影響
